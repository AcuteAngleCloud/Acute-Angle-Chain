13  /* The block log is an external append only log of the blocks. Blocks should only be written
14  * to the log after they irreverisble as the log is append only. The log is a doubly linked
15  * list of blocks. There is a secondary index file of only block positions that enables O(1)
16  * random access lookup by block number.
17  *
18  * +---------+----------------+---------+----------------+-----+------------+-------------------+
19  * | Block 1 | Pos of Block 1 | Block 2 | Pos of Block 2 | ... | Head Block | Pos of Head Block |
20  * +---------+----------------+---------+----------------+-----+------------+-------------------+
21  *
22  * +----------------+----------------+-----+-------------------+
23  * | Pos of Block 1 | Pos of Block 2 | ... | Pos of Head Block |
24  * +----------------+----------------+-----+-------------------+
25  *
26  * The block log can be walked in order by deserializing a block, skipping 8 bytes, deserializing a
27  * block, repeat... The head block of the file can be found by seeking to the position contained
28  * in the last 8 bytes the file. The block log can be read backwards by jumping back 8 bytes, following
29  * the position, reading the block, jumping back 8 bytes, etc.
30  *
31  * Blocks can be accessed at random via block number through the index file. Seek to 8 * (block_num - 1)
32  * to find the position of the block in the main file.
33  *
34  * The main file is the only file that needs to persist. The index file can be reconstructed during a
35  * linear scan of the main file.
36  */
